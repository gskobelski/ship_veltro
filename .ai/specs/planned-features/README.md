# Planned Features

Ten katalog jest miejscem startowym dla każdej nowej zmiany.

Zasady:
- najpierw opis, potem kod,
- jedna zmiana = jeden plik spec = jeden branch = jeden PR,
- pomysły "na później" też trafiają tutaj, zamiast znikać z kontekstu,
- po merge'u plik powinien zostać usunięty albo przeniesiony do archiwum.

Minimalna zawartość pliku:
- cel zmiany,
- zakres ekranów / endpointów / danych,
- flow użytkownika,
- edge case'y,
- uwagi wdrożeniowe.
